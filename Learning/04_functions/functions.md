# Functions: Reusable Blocks of Code

Functions allow you to group code together and give it a name. Instead of writing the same logic over and over, you just "call" the function.

### The Anatomy of a Function
A function needs three main things:
1. **Return Type**: What does the function give back? (`int`, `double`, `string`, or `void` if it gives nothing).
2. **Name**: What is it called? (e.g., `calculateDamage`).
3. **Parameters**: What information does it need to work? (e.g., `int strength`).



### Why use them?
* **Organization**: Keeps your `main()` function from getting too messy.
* **Reusability**: Write once, use everywhere.
* **Maintenance**: If you need to change how "Movement" works, you only change it in one place.
